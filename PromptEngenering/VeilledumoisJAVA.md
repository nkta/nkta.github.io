## Prompt Engineering pour la veille java du mois
```
Contexte et objectifs :
Je souhaite réaliser une veille Prompt Engineering complète sur les nouveautés, annonces et évolutions majeures du mois de mai. Mon but est de couvrir à la fois :

    Les mises à jour du JDK (nouvelles versions, correctifs, fonctionnalités en preview).

    Les évolutions et tendances autour des frameworks Java majeurs (Spring, Jakarta EE, Micronaut, Quarkus, etc.).

    Les nouvelles bibliothèques ou outils intéressants pour les développeurs Java (librairies pour la persistance, le test, l’observabilité, etc.).

    Les actualités liées à l’écosystème Java : conférences, articles techniques majeurs, annonces officielles d’Oracle, événements communautaires.

    Les meilleures pratiques apparues ou soulignées récemment dans la communauté Java (patterns, outils, conseils d’architecture).

Période visée :

    Le mois écoulé (exemple : si nous sommes en avril, couvrir les nouveautés de mars ; adapter la période en fonction de la date actuelle).

Format et détails attendus :

    Synthèse générale des faits marquants (2-3 paragraphes).

    Liste détaillée des nouveautés techniques et des avancées notables.

    Références et sources (liens vers articles officiels, blogs de développeurs reconnus, annonces sur le site d’Oracle, communiqués de projets open source).

    Conseils ou bonnes pratiques à retenir (par ex. adoption d’une nouvelle fonctionnalité, impact sur la maintenance ou la performance).

    Mises en perspective : comment ces évolutions peuvent influencer la roadmap d’un projet Java dans les prochains mois.

Spécificités de la recherche :

    J’aimerais avoir des exemples de code pour illustrer certaines fonctionnalités ou nouveautés importantes.

    Si possible, obtenir une courte analyse critique : quels changements sont réellement impactants, quelles innovations sont plus anecdotiques ?

    Mentionner les événements de la communauté Java (conférences, meetups ou enregistrements vidéos publiés le mois dernier).

Sortie souhaitée :

    Une réponse structurée avec titres, sous-titres, listes à puces et éventuellement un récapitulatif final.

    Le tout en français, avec un style clair et concis, mais approfondi techniquement.

Instruction supplémentaire :

    Ne pas oublier d’indiquer les versions de Java concernées (par exemple Java 17 LTS, Java 20, etc.) dans le cadre des nouveautés.

    S’il y a des roadmaps officielles (Spring, Jakarta EE…), fournir un résumé et la source.
```
